## I. THE PHILOSOPHY

### The Core Truth
Most productivity apps assume your brain is fully functional. Mindooo does not.

**The brain is a muscle — trainable, measurable, improvable.**

The Cognitive Performance System does not require artificial IQ tests or clinical assessments. Instead, it monitors cognitive operations **silently through natural usage** — every brain dump, every focus session, every chat interaction becomes data. From this data, the system builds a personalised brain training programme that adapts to Mo's specific strengths, weaknesses, and constraints.

### The Three Pillars

| Pillar | What It Means | How It Works |
|--------|---------------|--------------|
| **Monitor** | Silent observation of cognitive patterns | Real behaviour inference from chronicles, focus sessions, chat |
| **Train** | Targeted micro-exercises grounded in neuroscience | 5-10 minutes daily, adaptive difficulty, evidence-based |
| **Enhance** | Progressive improvement tracked over time | Weekly reports, trend analysis, personalised recommendations |

### Why This Matters for Mo

Mo has identified:
- **Dyslexia-type learning challenges** — needs multi-format input, no time pressure
- **ADHD-type execution challenges** — needs shorter sessions, visual anchors, frequent wins
- **Zero coding skills** — the system must be intuitive, never technical

**If the Cognitive Performance System works for Mo, it works for anyone.**

### The Mindooo Difference

| Traditional Cognitive Apps | Mindooo Cognitive Performance |
|---------------------------|-------------------------------|
| Artificial tests (IQ, memory games) | Real behaviour inference from natural usage |
| Generic training programmes | Personalised to your actual cognitive profile |
| One-size-fits-all exercises | Adaptive to learning difficulties and strengths |
| Static scores | Progressive tracking with trend analysis |
| Isolated from life context | Integrated with all Mindooo modules |

## III. THE COGNITIVE DIMENSIONS

The Cognitive Performance System monitors and trains **14 distinct cognitive dimensions**, organised into 4 domains.

### Domain 1: Memory Systems (4 Dimensions)

| Dimension | What It Is | Natural Measurement | Training Target |
|-----------|------------|---------------------|-----------------|
| **Working Memory** | Holding and manipulating information in real-time | Chronicle complexity (word count, theme count, structure depth); multi-step task tracking in chat | Capacity expansion, chunking strategies |
| **Short-Term Memory** | Brief retention of recent information | Chronicle recall accuracy in chat ("What did you dump 3 days ago?"); theme consistency across recent dumps | Encoding strategies, rehearsal techniques |
| **Long-Term Memory** | Durable storage of knowledge and experiences | Theme recurrence over weeks; vocabulary consistency; concept linking across chronicles | Spaced retrieval, elaborative encoding |
| **Episodic Memory** | Memory of personal experiences and events | Journal entry detail and temporal accuracy; personal event referencing in chat | Narrative coherence, event structuring |

### Domain 2: Attention Functions (3 Dimensions)

| Dimension | What It Is | Natural Measurement | Training Target |
|-----------|------------|---------------------|-----------------|
| **Sustained Attention** | Maintaining focus over extended periods | Focus session completion rate; average session duration; interruption recovery time | Focus stamina building, distraction resistance |
| **Selective Attention** | Filtering irrelevant stimuli | Distraction logging in focus sessions; task-switching frequency; notification response patterns | Stimulus filtering, priority maintenance |
| **Attentional Recovery** | Speed of focus restoration after interruption | Time between interruption and session resume; post-interruption completion rate; energy restoration patterns | Rapid refocusing, stress inoculation |

### Domain 3: Executive Functions (4 Dimensions)

| Dimension | What It Is | Natural Measurement | Training Target |
|-----------|------------|---------------------|-----------------|
| **Planning & Organisation** | Sequencing and structuring actions | Project Launcher engine usage quality (step completeness, logical flow); task list organisation | Sequencing skills, hierarchical planning |
| **Cognitive Flexibility** | Switching between concepts and perspectives | Theme diversity in brain dumps; reframing attempts in chat; multi-solution generation | Perspective shifting, adaptability |
| **Inhibition Control** | Suppressing irrelevant responses and impulses | Habit engine data (urge resistance, relapse patterns); impulse logging; delay-of-gratification instances | Impulse management, response suppression |
| **Working Memory Updating** | Holding and updating mental representations | Chat conversation depth and topic tracking; multi-thread reasoning; real-time information integration | Dynamic updating, mental juggling |

### Domain 4: Processing & Awareness (3 Dimensions)

| Dimension | What It Is | Natural Measurement | Training Target |
|-----------|------------|---------------------|-----------------|
| **Processing Speed** | Speed of information intake and response | Words per minute in brain dumps; dump session efficiency; chat response latency; decision speed | Rapid encoding, efficient processing |
| **Metacognition** | Awareness and regulation of one's own thinking | Self-assessment accuracy vs. actual performance; confidence calibration; blind spot identification | Self-monitoring, strategic awareness |
| **Cognitive Integration** | Combining multiple cognitive resources effectively | Cross-module usage patterns (brain dump → focus → chat coherence); holistic problem-solving | Resource coordination, synergistic thinking |

## IV. THE INFERENCE ENGINE

### The Core Principle
**All scores are 0-100. All inferred from real behaviour. No artificial tests required.**

The Inference Engine is a set of algorithms that convert natural usage patterns into quantitative cognitive scores. It runs silently in the background, updating scores after every significant interaction.

### Score Calculation Formulas

#### Attention Score (0-100)
```
BASE COMPONENTS:
  focus_completion_rate     × 35  // % of started sessions completed
  avg_focus_duration / 90   × 25  // 90 min = perfect (ultradian rhythm)
  (1 - distraction_rate)    × 20  // % of time without interruption
  recovery_speed_score      × 20  // speed of refocusing after break

CALCULATION:
  attention_score = MIN(100, ROUND(
    (focus_completion_rate × 35) +
    (MIN(avg_focus_duration, 90) / 90 × 25) +
    ((1 - distraction_rate) × 20) +
    (recovery_speed_score × 20)
  ))

WEIGHTED TREND:
  // Recent performance weighted more heavily
  weighted_score = (current_week × 0.6) + (previous_week × 0.3) + (two_weeks_ago × 0.1)
```

#### Memory Score (0-100)
```
BASE COMPONENTS:
  chronicle_recall_accuracy    × 30  // future: quiz on past chronicles
  vocabulary_consistency       × 25  // stable vocabulary = good semantic memory
  theme_consistency            × 20  // recurring themes = conceptual retention
  temporal_accuracy            × 15  // correct dating of personal events
  detail_richness              × 10  // depth of episodic encoding

CALCULATION:
  memory_score = MIN(100, ROUND(
    (chronicle_recall_accuracy × 30) +
    (vocabulary_consistency × 25) +
    (theme_consistency × 20) +
    (temporal_accuracy × 15) +
    (detail_richness × 10)
  ))

NOTE: chronicle_recall_accuracy requires future spaced-repetition feature.
  Until then, substitute with: theme_recognition_rate × 0.8
```

#### Processing Speed Score (0-100)
```
BASE COMPONENTS:
  words_per_minute_normalized  × 40  // normalised against personal baseline
  dump_complexity_score        × 30  // complex structures = faster processing
  response_latency_score       × 20  // speed of chat responses
  decision_speed_score       × 10  // time to choose in decision exercises

CALCULATION:
  processing_score = MIN(100, ROUND(
    (words_per_minute_normalized × 40) +
    (dump_complexity_score × 30) +
    (response_latency_score × 20) +
    (decision_speed_score × 10)
  ))

NORMALISATION:
  words_per_minute_normalized = MIN(1.0, current_wpm / baseline_wpm)
  // Baseline established from first 10 brain dumps
  // Score improves as user exceeds their own baseline
```

#### Cognitive Flexibility Score (0-100)
```
BASE COMPONENTS:
  theme_diversity_index        × 35  // variety of topics in brain dumps
  reframing_attempts_rate      × 25  // frequency of perspective shifts in chat
  multi_solution_generation    × 25  // number of solutions generated to problems
  task_switching_efficiency    × 15  // performance when switching between modules

CALCULATION:
  flexibility_score = MIN(100, ROUND(
    (theme_diversity_index × 35) +
    (reframing_attempts_rate × 25) +
    (multi_solution_generation × 25) +
    (task_switching_efficiency × 15)
  ))
```

#### Planning & Organisation Score (0-100)
```
BASE COMPONENTS:
  project_plan_quality         × 40  // completeness of Project Launcher outputs
  task_sequencing_accuracy     × 25  // logical ordering in self-generated plans
  goal_decomposition_depth     × 20  // granularity of goal breakdown
  deadline_adherence           × 15  // on-time completion of self-set deadlines

CALCULATION:
  planning_score = MIN(100, ROUND(
    (project_plan_quality × 40) +
    (task_sequencing_accuracy × 25) +
    (goal_decomposition_depth × 20) +
    (deadline_adherence × 15)
  ))
```

#### Inhibition Control Score (0-100)
```
BASE COMPONENTS:
  urge_resistance_rate         × 40  // habit engine: resisted urges / total urges
  impulse_delay_success        × 30  // delayed gratification instances
  distraction_ignore_rate      × 20  // ignored distractions / total distractions
  emotional_reactivity_score   × 10  // (inverse) speed of emotional escalation

CALCULATION:
  inhibition_score = MIN(100, ROUND(
    (urge_resistance_rate × 40) +
    (impulse_delay_success × 30) +
    (distraction_ignore_rate × 20) +
    ((1 - emotional_reactivity_score) × 10)
  ))
```

#### Metacognition Score (0-100)
```
BASE COMPONENTS:
  self_assessment_accuracy     × 40  // correlation between predicted and actual performance
  confidence_calibration       × 30  // appropriate confidence levels (not over/under)
  blind_spot_recognition       × 20  // acknowledging limitations when pointed out
  strategy_adjustment_rate     × 10  // changing approach based on feedback

CALCULATION:
  metacognition_score = MIN(100, ROUND(
    (self_assessment_accuracy × 40) +
    (confidence_calibration × 30) +
    (blind_spot_recognition × 20) +
    (strategy_adjustment_rate × 10)
  ))
```

#### Overall Cognitive Score (0-100)
```
WEIGHTED COMPOSITE:
  overall_score = ROUND(
    (attention_score         × 0.18) +
    (memory_score            × 0.16) +
    (processing_score        × 0.14) +
    (flexibility_score       × 0.12) +
    (planning_score          × 0.12) +
    (inhibition_score        × 0.10) +
    (metacognition_score     × 0.10) +
    (cognitive_integration   × 0.08)
  )

// Weights reflect relative importance for Mo's goals
// Attention and memory weighted highest (core deficits)
// Cognitive integration weighted lowest (advanced skill)
```

## VIII. LEARNING DIFFICULTIES SUPPORT

### The Adaptive Philosophy
Mo has identified learning difficulties. The system does not treat these as deficits to overcome, but as **neurodivergent patterns to accommodate and leverage**.

### ADHD-Type Support

| Challenge | System Adaptation | Science |
|-----------|-------------------|---------|
| Short attention span | Exercises capped at 3 minutes; frequent micro-breaks; visual progress indicators | Barkley's executive function model (1997) |
| Task initiation difficulty | "Start with 1 minute" option; gamified streaks; immediate feedback | Behavioural activation (Martell et al., 2010) |
| Distractibility | Single-task mode; distraction logging (not shaming); environment design prompts | Stimulus control (Powers et al., 2015) |
| Impulsivity | Urge surfing exercises; 10-second pause prompts; delayed gratification training | DBT skills (Linehan, 1993) |
| Emotional dysregulation | Mood check-ins before/after exercises; emotional tone tracking in chronicles | Emotion regulation (Gross, 1998) |
| Hyperfocus risk | Session timers with gentle alerts; transition prompts between exercises | Time management (Barkley, 2012) |

### Dyslexia-Type Support

| Challenge | System Adaptation | Science |
|-----------|-------------------|---------|
| Reading difficulty | Voice-first options; text-to-speech for all exercises; dyslexia-friendly font (OpenDyslexic) | Shaywitz (2003) |
| Writing difficulty | Voice input for all responses; reduced writing requirements; visual alternatives | Writing accommodation (Gregg, 2009) |
| Processing speed | No time pressure on inputs; extended time options; complexity-matched exercises | Processing speed research (Decker, 2020) |
| Memory retrieval | Visual mnemonics; spaced repetition with images; concept mapping | Dual coding theory (Paivio, 1986) |
| Sequencing difficulty | Step-by-step breakdowns; numbered sequences; visual flowcharts | Sequential processing (Berninger, 2004) |
| Phonological issues | Minimal phonological tasks; emphasis on semantic and visual processing | Phonological deficit theory (Ramus, 2003) |

### General Accessibility

| Feature | Implementation | Benefit |
|---------|------------------|---------|
| **Multi-format input** | Text, voice, drawing, photo for every exercise | Matches preferred input style |
| **Adjustable difficulty** | User can override AI suggestion; always has control | Autonomy and self-determination |
| **Progress celebration** | Explicit recognition of small wins; streak milestones | Positive reinforcement and motivation |
| **No failure states** | Exercises are practice, not tests; no scores on individual exercises | Growth mindset preservation |
| **Contextual help** | "Why this exercise?" button with science explanation | Understanding enhances engagement |
| **Pacing control** | Pause, resume, save-for-later on all exercises | Self-regulated learning |

### Functions

```sql
-- ============================================================
-- FUNCTION: calculate_cognitive_overall
-- Recalculates overall score from dimension scores
-- ============================================================
CREATE OR REPLACE FUNCTION calculate_cognitive_overall(p_user_id uuid)
RETURNS integer
LANGUAGE plpgsql
STABLE
SECURITY DEFINER
AS $$
DECLARE
  v_overall integer;
BEGIN
  SELECT ROUND(
    (COALESCE(attention_score, 0) * 0.18 +
     COALESCE(working_memory, 0) * 0.08 +
     COALESCE(short_term_memory, 0) * 0.04 +
     COALESCE(long_term_memory, 0) * 0.04 +
     COALESCE(episodic_memory, 0) * 0.04 +
     COALESCE(focus_stamina, 0) * 0.09 +
     COALESCE(selective_attention, 0) * 0.05 +
     COALESCE(attention_recovery, 0) * 0.04 +
     COALESCE(planning_score, 0) * 0.12 +
     COALESCE(cognitive_flexibility, 0) * 0.12 +
     COALESCE(inhibition_control, 0) * 0.10 +
     COALESCE(working_memory_update, 0) * 0.04 +
     COALESCE(processing_speed, 0) * 0.14 +
     COALESCE(metacognition, 0) * 0.10 +
     COALESCE(cognitive_integration, 0) * 0.08
    )::numeric
  )::integer
  INTO v_overall
  FROM cognitive_profile
  WHERE user_id = p_user_id;

  RETURN COALESCE(v_overall, 0);
END;
$$;

GRANT EXECUTE ON FUNCTION calculate_cognitive_overall TO authenticated;

-- ============================================================
-- FUNCTION: get_brain_gym_streak
-- Calculates current consecutive days with brain gym sessions
-- ============================================================
CREATE OR REPLACE FUNCTION get_brain_gym_streak(p_user_id uuid)
RETURNS integer
LANGUAGE plpgsql
STABLE
SECURITY DEFINER
AS $$
DECLARE
  v_streak integer := 0;
  v_current_date date := CURRENT_DATE;
BEGIN
  LOOP
    PERFORM 1 FROM brain_gym_sessions
    WHERE user_id = p_user_id
      AND session_date = v_current_date
    LIMIT 1;

    IF FOUND THEN
      v_streak := v_streak + 1;
      v_current_date := v_current_date - 1;
    ELSE
      EXIT;
    END IF;
  END LOOP;

  RETURN v_streak;
END;
$$;

GRANT EXECUTE ON FUNCTION get_brain_gym_streak TO authenticated;
```
